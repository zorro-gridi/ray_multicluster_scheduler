# 一，规范说明
代码开发规范是进行程序开发必须遵守的行为准则

# 二，约束范围
本规范对项目的所有开发工作都具有约束力，请严格执行

# 三，开发环境
- 1. 使用“k8s” conda虚拟环境执行python代码
- 2. 使用符合MacOS的命令行指令

# 四，设计规范
## 4.1 模块化设计
- 1. 使用结构化设计方法进行功能开发，提高代码复用率
- 2. 使用面向对象的设计方法进行模块开发，简化模块接口调用

## 4.2 编码规范
- 1. 接口设计必须包含完整的函数签名、文档字符串、类型注解、并遵守 PEP8 编码规范
- 2. 仔细分析接口功能设计的传参需求，准确使用位置参数、与关键字参数等不同参数类型。谨慎使用关键字参数，避免参数注入风险
- 3. 杜绝使用硬编码，功能参数配置请合理使用数据类或配置文件管理

## 4.3 代码审查
- 1. 滥用参数默认值。防止由于函数参数默认值掩盖调用方遗漏参数，导致接口行为异常或语义漂移
- 2. 数据模型传参完整性与一致性。防止数据模型转换过程中，丢失关键参数，保证参数传递链路的完整性与一致性

# 5. 日志管理
## 5.1 日志打印规范
- 1. 控制台信息打印格式: <Module>.<SubModule>... - <message>
- 2. 关键数字信息使用合适的 emoji 图标标注
- 3. 字典、列表等数据类型，使用 pprint 友好输出
